## 사용법

- 원격 저장소를 로컬 저장소에 연동 = Clone
- 변경사항(작업사항) 확인 후 커밋 = Commit
- 원격 저장소에 변경사항 반영 = Push
- 다른 사람의 작업 내역이 있는지 확인 = Fetch
- 다른 사람의 작업 내용을 내 PC에 반영함 = Pull

### 주요 명령어 (<mark>중요</mark>)

- Clone : Git 원격 저장소를 로컬 저장소에 연결
- Commit : 변경사항을 묶어서 기록.
- Push : 원격 저장소에 변경사항(커밋 이력) 반영 (밀어넣기)
- Fetch : 원격 저장소에 변경사항이 있는지 확인
- Pull : Fetch 결과 변경사항이 있으면 로컬에 변경사항을 적용 (당겨오기)

> `Fetch`는 숨쉴때마다 한다.


## 어떤 타이밍에 커밋을 해야 하는지?

- 구현해야 할 작은 기능 하나 단위로 커밋
	- 능숙해지면 가능.
	- 초심자 입장에서 어떤 파일을 건드려야 기능이 완성될지 감 잡기가 힘듬.
	- 현 단계에서는 특정 파일 수정하면 그냥 커밋.
		- ex> 플레이어 이동기능 구현 중이었다면
			- [Feat] 플레이어 이동 구현을 위한 CharacterController 수정
			- [Feat] 플레이어 이동 구현을 위한 CharacterMovement 수정
			- [Feat] Controller에서 Movement 제어하도록 수정

### 클래스룸을 통해서 생성된 레포지토리에는 언제 커밋?

- 쉬는시간마다 커밋
- 오전 / 오후 끝날때 커밋
- 과제 제출한 내용중에 이후에 프로젝트에 써먹을 수 있을 것 같은 스크립트들 추가하고 커밋
